## 积分差异根因

旧 SQL 的积分口径是：

```sql
SUM(pt.fz_point)
SUM(pt.yy_point)
```

再在展示层或对比时取整，因此全省出现：

| 口径 | SQL fzPoint | Excel fzPoint | SQL yyPoint | Excel yyPoint | SQL sumPoint | Excel sumPoint |
|---|---:|---:|---:|---:|---:|---:|
| 地市汇总原始小数后取整 | 47106 | 47101 | 15812 | 15812 | 62918 | 62913 |

反推后确认，下半表使用的是：

```sql
SUM(ROUND(pt.fz_point, 0))
SUM(ROUND(pt.yy_point, 0))
```

即：

```text
积分明细单条先四舍五入成整数，再按地市和全省汇总。
sumPoint = fzPoint + yyPoint。
```

按该口径后：

| 口径 | SQL fzPoint | Excel fzPoint | SQL yyPoint | Excel yyPoint | SQL sumPoint | Excel sumPoint |
|---|---:|---:|---:|---:|---:|---:|
| 明细单条取整后汇总 | 47101 | 47101 | 15812 | 15812 | 62913 | 62913 |

## 结构差异

SQL 输出比 Excel 下半表多 1 行：

| area_no | area_name | 说明 |
|---|---|---|
| 530 | 菏泽 | SQL 有菏泽明细；Excel 下半表未展示菏泽明细，但全省汇总包含菏泽 |

这不是数值口径差异，而是展示行范围差异。

## 结论

积分上的 diff 已解决。根因不是账期，也不是人员分区，而是积分展示/聚合规则：

```text
acct_day = T
p_acct_day = T+1
有效装维人员过滤保持不变
积分明细单条 ROUND 后再汇总
```

## 积分差异原因

Excel 本文件的积分更接近原始小数直接汇总，而不是明细单条 ROUND 后汇总。

全省对比：

| 口径 | 发展积分 | 运营积分 | 原始/价值积分 |
|---|---:|---:|---:|
| Excel | 46169.4 | 17575.8 | 63745.2 |
| `p_acct_day=20260709` 原始小数汇总 | 46169.43 | 17575.8 | 63745.23 |
| `p_acct_day=20260709` 明细 ROUND 后汇总 | 46166 | 17571 | 63737 |

当前 `score=T.sql` 写法是：

```sql
SUM(ROUND(COALESCE(TRY_CAST(pt.fz_point AS DOUBLE), 0), 0)) AS fz_point,
SUM(ROUND(COALESCE(TRY_CAST(pt.yy_point AS DOUBLE), 0), 0)) AS yy_point
```

因此会相对 Excel 少：

```text
发展积分少 3.4
运营积分少 4.8
原始/价值积分少 8.2
```

结论：

```text
20260709_各地市随销统计.xlsx 使用 score=T 分区，积分展示保留原始小数汇总；
不要套用 20260702 下半表“明细单条 ROUND 后汇总”的口径。
```
