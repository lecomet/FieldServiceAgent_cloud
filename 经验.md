# 装维随销统计经验

## 先判断文件版本和 score 口径

装维随销统计的 Excel 不是所有日期都共用同一套积分展示口径。

已验证：

| 文件 | 积分分区 | 积分展示 |
|---|---|---|
| 随销统计0702账期-20260706.xlsx 下半表 | `acct_day=T`，`p_acct_day=T+1` | 明细单条 ROUND 后汇总 |
| 20260709_各地市随销统计.xlsx | `acct_day=T`，`p_acct_day=T` | 原始小数直接汇总 |

经验：

```text
文件名或 SQL 名里出现 score=T / score=T+1 时，先按文件指定 score 口径跑；
不要把某一天验证出的积分 ROUND 规则直接套到所有文件。
```

## 业务量优先验全省合计

20260709 对比中，移动、宽带、FTTR、FTTR-B、151、天翼智屏的全渠道和装维量全省全部对齐。

经验：

```text
如果业务量全省合计已经 0 差异，不要先改业务关联、日期、分区。
继续查人数、积分、展示字段映射。
```

## 人数差异优先查实习 cutoff

20260709 中，当前 SQL 使用：

```text
intern_start_time <= T_DATE - 3个月 = 2026-04-09
```

得到：

```text
正式 2829 + 实习 22 = 2851
```

Excel 总人数为 2961。把实习 cutoff 放宽到 `2026-06-09` 后：

```text
正式 2829 + 实习 138 = 2967
```

与 Excel 只差 6。

经验：

```text
总人数大幅差异时，不要只盯正式人员 attr_value 或白表岗位；
先查实习人员 intern_start_time 分布，特别是是否存在某一天集中入职的大批人员。
```

继续验证“实习一年内”时发现：

```text
一年内到 T=2026-07-09：正式+实习=3055，比 Excel 2961 多 94。
一年内到月初 2026-07-01：正式+实习=3036，比 Excel 2961 多 75。
```

所以“实习一年内”不是 20260709 Excel 的总人数口径。

更贴近的候选口径是：

```text
实习人员取 is_confirm=0 且 intern_start_time <= 2026-06-20：
正式 2829 + 实习 132 = 2961，和 Excel 全省总人数一致。
```

但该候选口径逐地市还有小差异，不能直接写死为正式规则。

后续按业务提示继续验证：

```text
实习人员需要同时满足 ext1=2 和 zw_type=新装维。
```

对应 SQL 口径：

```sql
emp.ext1 = '2'
AND emp."month" = '202607'
AND emp.p_acct_day = '20260709'
AND emp.intern_start_time <= '2026-06-09'
AND e.zw_type = '1'
```

地市和人员唯一键仍按原实习规则：

```text
地市归属：emp.ext7
人员唯一键：emp.ext18
```

验证结果：

```text
正式人员 2829
实习人员 138
合计 2967
Excel 2961
差异 +6
```

逐地市只剩：

```text
威海 +3
淄博 +1
临沂 +1
东营 +1
```

经验：

```text
20260709 总人数差异的主因不是“一年内”，而是实习人员需叠加 zw_type=新装维；
“实习一个月”按 T-1月日期 2026-06-09 更贴近，按月初 2026-06-01 会少 81 人。
```

继续排查后，`intern_start_time` 不是关键过滤。对 20260709 来说：

```sql
emp.ext1 = '2'
AND emp."month" = '202607'
AND emp.p_acct_day = '20260709'
AND emp.zw_type = '1'
```

不加 `intern_start_time` 与加 `intern_start_time <= '2026-06-09'` 结果一样，都是：

```text
实习人员 138
正式 2829 + 实习 138 = 2967
Excel 2961
差异 +6
```

经验：

```text
20260709 实习人员优先按 ext1=2 + zw_type=1 标签判断；
intern_start_time 不能作为当前 Excel 对齐的主要依据，旧 SQL 里的三个月 cutoff 不适合这份 Excel。
```

剩余 6 人不是简单字段能过滤掉：

```text
威海 +3
淄博 +1
临沂 +1
东营正式人员已多 +1
```

已验证无效或不宜直接采用的过滤：

```text
is_confirm=0：实习人数从 138 降到 122，会少太多。
一年内：会查多到 226 个实习。
intern_start_time 04-01 到 04-09：只有 21/22 个实习，少 110 左右。
```

注意：

```text
这 138 个实习人员基本无法有效关联到 dim_staff_zw10_zw 的 staff_num/attr_value/area_no；
实习人员地市归属应继续用 emp.ext7，人员唯一键用 emp.ext18。
```

剩余 6 人排查经验：

```text
如果 Excel 只有地市汇总，没有人员明细，无法直接判断某个人是否在 Excel 里；
只能把差异缩小到候选城市和候选人员，再向源系统/明细名单核对。
```

20260709 当前候选范围：

```text
淄博：5 个实习里应少 1 个
临沂：33 个实习里应少 1 个
威海：6 个实习里应少 3 个
东营：82 个正式里应少 1 个
```

另有 8 个 `ext1=2 + zw_type=1` 实习人员 `emp.ext7` 为空。Excel 没有空地市行，因此不能仅凭汇总表判断这 8 人是否被剔除或被源系统补归属。遇到这种情况，必须拿人员明细按 `emp.ext18`/姓名核对。

用 `个人随销统计_20260701.xlsx` 辅助核查时要注意日期差异：

```text
0701 个人表不是 0709 明细，只能作为参考。
```

本次对比结果：

```text
实习候选按 emp.ext18 对个人表“工号”匹配：0 人；
实习候选按姓名匹配：11 行，存在同名风险；
8 个空地市实习中，只有“张进”按姓名匹配到德州-武城 AGL1702309，但工号不同，不能确认同一人；
东营 82 个正式候选均能按工号匹配到 0701 个人表。
```

经验：

```text
跨日期个人表不能作为最终剔除依据；
工号/人员键匹配优先于姓名匹配；
姓名匹配只能提示线索，不能定规则。
```

## attr_value=200 不一定影响人数

20260709 人数复核中，把正式人员过滤从：

```sql
zw.attr_value IN ('10','20','50','72','73')
```

放宽为：

```sql
zw.attr_value IN ('10','20','50','72','73','200')
```

人数没有变化。

经验：

```text
装维门店 attr_value=200 在业务装维量过滤中要保留；
但人数差异是否由 200 引起必须用数据验证，不要默认判断。
```

## 积分小数差异看是否 ROUND

20260709 全省积分：

```text
Excel 原始积分：63745.2
p=20260709 原始小数汇总：63745.23
p=20260709 明细 ROUND 汇总：63737
```

经验：

```text
积分差异只有个位数时，优先检查“明细 ROUND 后汇总”还是“原始小数汇总”；
这类差异不一定是账期或人员过滤问题。
```
